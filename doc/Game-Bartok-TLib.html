class

class Ruleable a where Source #

A class for types that are smaller than Rule so that there are different options for how to convert.

Be careful when composing these. doBefore a . doBefore b == doBefore (b . a) doAfter a . doBefore b == doBefore (a . b)

Check the source code of TSample to see how to use these.

Also be careful when both reading and writing variables. See rAce for an example of how to set and unset variables to get a oneshot effect without missing

type Game = Event -> Step Source #

This is the type of games (meaning a ruleset, rather than a specific instance of a game). Given an event such as a player action, and a game state, it returns the resulting state

type Rule = Game -> Game Source #

The type of rules - a rule is added by function composition forming a chain such as 'r3 (r1 (r2 baseAct))' . Most rules should call the first argument under most circumstances and usually without changing the event.
